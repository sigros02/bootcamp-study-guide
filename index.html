HTML

  • head tag is invisible to the user and defines how information is handled by the client for this web page
  • meta tag is invisible to the user and holds variables and values that are used by the client for this webpage
  • title tag is sometimes used to define the search engine tab name which is visible to the user. More importantly it is used for search engine optimization
  • Here is a good reference link for Markdown language syntax

CSS

  • INLINE CSS: Styles element directly in HTML file. Applies to specific HTML tag. Consists of a "style" attribute followed by a CSS rule.
  • INTERNAL STYLE SHEET: Contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that webpage.
  • EXTERNAL STYLE SHEET: Links a file using "link" element containing CSS rules separately. Makes design changes easier.
  • * is a "wild card" selector that applies all properties in the given declaration box to all visible elements on the page.
  • You can combine multiple selector separated by commas who will share the same properties within a proceeding declaration box.
  • Class attributes can be used as CSS selectors instead of HTML elements. Then the class attribute can be assigned to specific elements within HTML.

Git

  • GIT is an SCM or Source Code Manager and is a command line tool
  • Here is a good reference link for commonly used git commands
  • Here is a good reference link for commonly used Windows terminal commands
  • The .git folder in your repo contains information for all previous commits
  • Local git repo is stored on your local computer and is updated every time you "save" your work in your IDE
  • git commit command takes your local copy and moves any changes in your commit to your .git folder
  • git push command copies changes to the server or "origin" where others can see

JavaScript

  • Truthy values are anything NOT null, undefined, false or zero
  • Functions can be exited via return statement, throw an exception, or reaching the end of a function
  • If the end of a function is reached without a return or exception, then the function will return undefined
  • Objects or classes are similar to nouns and they can have attributes and/or actions
  • Functions or methods are similar to verbs since they "do stuff", aka actions
  • Properties are similar to adjectives since they "describe stuff", aka attributes
